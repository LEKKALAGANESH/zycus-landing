# SEO Content Plan

This document specifies the on-page and technical SEO contract for the Zycus landing deliverable. It covers four public URLs — Home (`/`), Thank You (`/thank-you/`), Privacy (`/privacy/`), and Terms (`/terms/`) — with Yoast SEO Free as the recommended plugin on the WordPress build and hand-emitted `<meta>` + JSON-LD on the PHP reference build. Brand voice is refined B2B, targeting procurement leaders evaluating agentic AI for Source-to-Pay.

## Primary keyword targets

Ordered by proximity to a demo-request conversion (decision intent first):

- `zycus merlin AI demo` — decision. Branded, bottom-funnel; visitor already knows the product.
- `AI source-to-pay platform` — decision. Category-defining buyer query at shortlist stage.
- `agentic AI procurement` — consideration/decision. Primary hero keyword; high intent, rising volume.
- `automated sourcing and contracting` — consideration. Solution-aware buyer comparing modules.
- `invoice automation AI` — consideration. Narrow workflow query mapping to the AP use case.
- `procurement automation software` — consideration. Broad category term; competitive but essential.
- `enterprise procurement software` — awareness. Top-of-funnel; use for secondary headings and alt text.

## Structured data

JSON-LD is emitted server-side in the `<head>`; no client hydration. Schemas in scope:

- `Organization` — global, every page. Fields: `name`, `url`, `logo` (absolute URL to `/assets/img/zycus-logo.webp`), `sameAs[]` listing LinkedIn and Twitter/X handles. Drives the Knowledge Panel eligibility.
- `FAQPage` — Home only. Auto-generated from the four Q/A pairs in `config/faqs.php` (PHP build) or mirrored from the Elementor Accordion widget (WP build). Must validate clean in Google Rich Results Test before each release.
- `WebSite` with `potentialAction` SearchAction — optional, flagged for v2 once a site search is added. Skip for v1.
- `BreadcrumbList` — not applicable on a single-page landing; omit.

Minimal `FAQPage` JSON-LD shape (one Q/A shown):

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "How quickly can Zycus be deployed?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "Most enterprise customers go live within 12-16 weeks using our phased S2P rollout."
    }
  }]
}
```

## OpenGraph and Twitter

Per-page social metadata:

- `og:title` and `og:description` mirror the meta title and description from the per-page table above — never diverge.
- `og:image` points to `/assets/img/zycus-og.jpg` at 1200x630. The logo at `/assets/img/zycus-logo.webp` is the current fallback.
- `og:type` is `website` for `/`, `article` for `/privacy/` and `/terms/`.
- `og:url` uses the canonical absolute URL.
- `twitter:card` is `summary_large_image`; `twitter:site` is the brand handle.

Flag: **create a proper 1200x630 OG image before launch**. The current brain-logo fallback is 160x160 and renders as a blurred thumbnail in LinkedIn and Slack unfurls — unacceptable for B2B share quality.

## Technical SEO checklist

- `<html lang="en">` on every page.
- Canonical URL emitted in `<head>` using `APP_URL` + `REQUEST_URI`; strips query strings.
- `sitemap.xml` auto-generated by Yoast (WP) or a hand-written sitemap served from `/sitemap.xml` (PHP).
- `robots.txt` present at `/robots.txt`; `Allow: /`; points to the sitemap.
- Heading hierarchy: exactly one H1 per page, no skipped levels (H2 -> H3 only).
- Image alt text on every content image; brand logo alt is exactly `"Zycus AI Procurement Brain Logo"`.
- Inter font preloaded (`<link rel="preload" as="font" crossorigin>`) to save ~100 ms LCP.
- Core Web Vitals pass — LCP < 2.5 s, CLS < 0.1, INP < 200 ms, measured on throttled 4G.
- Mobile-first indexing ready; the site is responsive by default.

## Content freshness plan

The home page is evergreen, but the `FAQPage` schema and trust-bar testimonial metrics must stay current to retain rich-result eligibility. Commit to refreshing testimonial numbers quarterly and reviewing the FAQ copy every six months; any edit should be accompanied by a rebuild of the JSON-LD block and a re-validation in the Rich Results Test. Owner: marketing ops, with engineering on-call for the deploy.
